# LIVIA 引き継ぎ書（現状共有 / AI補助用）

## 0. この文書の目的
このリポジトリは、創作世界「LIVIA（リヴィア）」の設定を **Markdownで一元管理**し、
- 自分（オーナー）が軸を決める
- 他メンバーが提案（PR/Issue）する
- オーナーがレビューして採用・統合する
という **“世界設定版 OSS”** 的な運用をするための土台。

また、AI補助にも併用して使うので、
「何が確定で、何が未確定か」「どのファイルを更新すべきか」を明確にする。

## 2. 確定している“世界の核”（現時点の公式）
現状、確定しているのは「骨格（コア設定）」のみ。

### 2.1 起源（世界の根幹）
- 起源と最終テーマの詳細は、ネタバレ防止のため非公開管理とする。
- 公開版では「感情と魔法の関係」「文明制約」「歴史構造」を主軸として提示する。

### 2.2 魔法体系
- 魔法は **感情を魔力に変換**して発動する。
- 感情の種類によって得意魔法が変わる。
- 個人の感情履歴（体験の積み重ね・偏り）から **固有魔法**が生まれる。

### 2.3 精霊
- 精霊は **感情の化身**。
- 契約によって魔法の使用や強化が成立する。
- 精霊は人間に対して「ミトコンドリア的な共生（エネルギー供給を支える）」という比喩で説明される。

### 2.4 文明（中世風に止まる理由）
- 魔法と電気エネルギーは干渉し合うため、電気文明が発展しづらい。
- その結果、世界の文明観は中世風に留まっている。

### 2.5 歴史スケール
- 歴史は約1万年規模。
- 周期的に大きな物語／戦争／転換点が起こる設計。
- 2000年ごとに主人公格とテーマを配置する構想がある（詳細未記入）。

## 4. 運用方針（コミュニティ参加型にする）
コンセプト：**オーナーが軸を守り、参加者が幅を増やす。**

### 4.1 提案のルール（参加者向け）
- 変更は原則 Issue で提案 → PR で反映
- 追加設定には必ず
  - どの“核設定”と整合するか
  - どのファイルに追記するか
  - 既存ルールを壊さない理由
を添える。

### 4.2 オーナーレビューの観点
- 世界の核（感情→魔力 / 非公開中核設定 / 中世停滞理由）と矛盾しないか
- “世界がリアルに暮らせる”方向に解像度が上がるか
- 設定が増えても読み手が迷子にならないか（索引化・用語統一）

## 5. AI補助にやらせたい仕事（具体）
AI補助には「発散・整理・フォーマット化」を任せ、
“決定”はオーナーがやる。

### 5.1 AI補助が得意なタスク
- TODOを分解してチェックリスト化（決める項目を洗い出す）
- 既存ファイルの内容から整合性チェック（矛盾候補の指摘）
- 年表テンプレの生成（1万年を2000年×5章に区切る枠組み）
- 魔法分類表の提案（ただし核設定は維持）
- 用語集（glossary）・索引・リンク構造の整備

### 5.2 AI補助へ投げるプロンプト例
- 「このリポジトリのTODOを抽出して、優先順位付きタスクリストにして」
- 「history_timeline.md を、2000年×5章のテンプレに整形して。各章に“必要項目”欄を作って」
- 「magic_system.md を読んで、矛盾しない範囲で“魔法系統ツリー案”を3案出して」
- 「精霊契約の儀式と契約破棄のルール案を、核設定（感情の化身）に沿って作って」

## 6. 次に人間（オーナー）が決めるべき最短ルート
人が来ない問題を解決するには、まず“見せられる核”を固める必要がある。
優先度順はこれ：

1. **1枚で伝わる世界の魅力**（world_overviewの強化 + キャッチコピー）
2. **魔法の具体例**（固有魔法の例を3〜10個）
3. **2000年ごとの5章立て（仮）**（主人公格は名前未定でも役割とテーマだけ先に決める）
4. **参加者が投稿しやすいテンプレ**（キャラ案テンプレ／地名案テンプレ／精霊案テンプレ）

## 7. 重要：現状の制約（誤解防止）
- 現在の内容は「オーナーが過去に語った“骨格”のみ」。
- 具体（国名・地理・キャラ詳細・事件）はまだ確定していない。
- ここからは、骨格を守りつつ、参加型で肉付けしていく段階。
