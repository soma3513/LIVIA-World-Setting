# 共同制作フロー（GitHubレビュー方式）

ぴっぴさんのイメージ：
- みんなが提案を投げる（Pull Request / Issue）
- ぴっぴさんがレビューする
- 採用したものだけが世界に組み込まれる（mainへマージ）

## 1. 参加導線（最小）
1. まず `README.md` と `world_overview.md` を読む
2. 追加したい要素を Issue で提案
3. 形にできる人は PR で追加（Markdown）

## 2. ルール（ブレ防止）
- 公開ルール（感情→魔力／中世風文明）と矛盾しない
- 非公開の中核設定に抵触しない（最終判定はオーナー）
- “強すぎる設定” を避ける（全能・無限・矛盾する例外を乱発しない）
- 追加した設定は「世界の他要素にどう影響するか」までセットで書く
  - 例：新しい魔法 → 社会／戦争／倫理にどう効く？

## 3. 提案テンプレ（Issue用）
- 追加したい要素：
- 目的（なぜ必要？）：
- コアへの整合性（感情・精霊・文明との関係）：
- 具体（誰が/どこで/いつ/どう使う）：
- リスク（矛盾しそうな点）：
- 関連ファイル（追記候補）：

## 4. 採用基準（例）
- 世界の“住めるリアリティ”が増えるか
- 既存ルールと矛盾しないか
- 他のクリエイターが二次創作しやすい余白があるか
